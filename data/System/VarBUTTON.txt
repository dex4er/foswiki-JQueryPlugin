#VarBUTTON
---+++ BUTTON{"text" ...} -- renders a nice button 
   * Parameters: 
     | *Parameter:* | *Description:* | *Default:* |
     | "text", value="text" | text to be put on this button | |
     | accesskey | access key used for this button | | 
     | class | extra class: use =foswikiRight= or =foswikiLeft= to specify aligment; \
               use =cyan=, =red=, =green= for different background colors; \
               use =simple= for a non-3d-button | | 
     | href | url of the click target | # | 
     | icon | icon to be put on the left; note, this can be any icon attached to the  ={IconSearchPath}=; see also VarJQICON | | 
     | id | html id for this button | | 
     | onclick | javascript event triggerd when clicking the button | | 
     | onmouseout | javascript event triggerd when the pointer leaves the button | | 
     | onmouseover | javascript event triggerd when the pointer hovers over the button | | 
     | target | topic to open when clicking on the button | | 
     | title | popup title displayed when hovering over the button | | 
     | type | type of action to be performed; available actions are \
              <ul>\
                <li>button: (default) normal click button, target specified in =target= or =href= parameter </li>\
                <li>clear: clears all input fields in a form this button is contained in</li>\
                <li>reset: resets all input fields of a form to their initial value </li>\
                <li>submit: submits the form this button is contained in</li>\
                <li>save: same as =submit= but takes care of extra validation steps when savign a wiki topic </li>\
              </ul> | button | 
   * Example: <div>%BUTTON{"%MAKETEXT{"Submit"}%" click="confirm('Are your sure?')"}%%BUTTON{"%MAKETEXT{"Cancel"}%" icon="cross" target="%WEB%.%TOPIC%"}% %CLEAR%</div>
   * Related: JQueryButton
   * Note: BUTTONS are floating to the left by default. Take care to add a =%<nop>CLEAR%= so that further content does not overlaoverlap with the button.
